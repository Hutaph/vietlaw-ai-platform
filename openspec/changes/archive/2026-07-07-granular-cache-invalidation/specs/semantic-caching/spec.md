## MODIFIED Requirements

### Requirement: Semantic cache storage and retrieval
The system SHALL use a dedicated Qdrant collection named `semantic_cache` (or similar storage mechanism) to store cached responses. The collection MUST store the vector representation of the rewritten query, and a payload containing the original query, response text, serialized citation context, timestamp, and an array `retrieved_doc_ids` containing the IDs of all documents retrieved to generate the response.

#### Scenario: Successfully caching a new response
- **WHEN** a cache miss occurs and a new response is successfully generated by the LLM
- **THEN** the system writes a new vector record to the `semantic_cache` collection containing the rewritten query vector and necessary metadata payload, including the `retrieved_doc_ids` array
